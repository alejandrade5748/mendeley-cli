## Reporting a Vulnerability

Use this section to tell people how to report a vulnerability.

Tell them where to go, how often they can expect to get an update on a
reported vulnerability, what to expect if the vulnerability is accepted or
declined, etc.

**Preferred: GitHub Security Advisories**

Open a private security advisory on this repository so it is only visible
to the maintainers:

https://github.com/VictorTomaili/mendeley-cli/security/advisories/new

**Alternative: Email**

If you cannot or prefer not to use GitHub Security Advisories, you can
email the maintainer at <victor@tomaili.com>. Please include the word
"security" or "vulnerability" in the subject line and encrypt sensitive
details with the maintainer's PGP key when possible.

**What to expect**

- An initial acknowledgement within **7 days** of the report.
- A status update (including triage outcome and planned fix) within
  **30 days**.
- If accepted: a fix will be prepared, a CVE will be requested when
  applicable, and the fix will be released in a patch version of the
  supported line. You will be credited in the advisory unless you ask
  to remain anonymous.
- If declined: a clear explanation of why the report is not considered
  a vulnerability in this project.

Please **do not** file public GitHub issues for suspected security
vulnerabilities.
